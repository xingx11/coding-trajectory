# Qwen / Claude Rubrics Templates

These are the working manual-scoring templates for the 10 Coding Trajectory bidding samples.

They follow the example files in `docs/examples/`:

- 10 Qwen TOML files under `qwen/`
- 10 Claude TOML files under `claude/`
- 7-10 criteria per file (AI selects from 20 candidates during `gen` or `score`)
- `points = 5` for every criterion
- `weight = 1.0` for most criteria (`architecture_boundaries_and_security_compliance` = 2.0)
- `passrate = sum(score × weight) / sum(points × weight)`

## Where The Fields Come From

- `name`: the scoring-dimension identifier, selected from 20 candidates by AI based on task characteristics.
- `description`: a task-specific scoring standard generated by AI, incorporating the project name, tech stack, and specific task details into 1-5 score tier definitions. Generic template descriptions are not acceptable.
- `type`: fixed as `likert` because each criterion is scored on a 1-5 scale.
- `points`: fixed as `5`, matching the provided examples.
- `weight`: fixed as `1.0` for most criteria (`architecture_boundaries_and_security_compliance` = 2.0).
- `score`: filled by AI scoring or manually after reading the trajectory.
- `rationale`: filled with concrete evidence for the score.

The `name` and `description` fields are auto-generated by AI during the `gen` stage (customized for each task's project and requirements). The `score` and `rescore` commands can further refine them using trajectory data.

## Fill-In Rules

- Only fill `score` and `rationale`.
- Keep `name`, `description`, `type`, `points`, and `weight` unchanged unless the requester provides an official replacement rubric.
- Use integer scores from `1` to `5`.
- Write a concrete rationale based on the actual trajectory, code changes, tests, and final answer.
- Do not submit templates where all scores remain `0`.

## Score Guide

- `5`: Fully satisfies the criterion with clear validation or evidence.
- `4`: Mostly satisfies the criterion with only minor omissions.
- `3`: Partially satisfies the main flow but has clear gaps, edge-case issues, or weak validation.
- `2`: Surface-level attempt; important requirements are missing.
- `1`: Almost no effective completion for this criterion.
- `0`: Not attempted or template not filled (unscored).

## Calculate Pass Rate

```powershell
python rubrics_templates\calc_passrate.py rubrics_templates\qwen rubrics_templates\claude
```

For filled files, the result should be interpreted as:

- Qwen target: `< 0.7`
- Claude target: `> 0.7`
- Claude should be higher than Qwen for the same task.
